# Claude 新项目前置阶段对话脚本模板

## 文档目的

这份模板适合：

- 从 0 开始的新项目
- 空白项目
- 还没有明确需求边界、方案和任务规划的项目

核心目标：

在真正开始写代码之前，让 Claude 先帮你完成：

1. 需求澄清
2. MVP 收敛
3. 方案比较
4. 任务规划
5. `CLAUDE.md` 初稿
6. 最小 skills / agents 建议

---

## 使用原则

- 一次只推进一个阶段
- 不要一上来就让 Claude 写代码
- 先把问题想清楚，再进入实现

最稳的顺序：

1. 需求澄清
2. MVP 边界
3. 方案比较
4. 任务规划
5. 项目规则
6. 协作框架

---

## 第一轮：需求澄清

直接复制：

```text
我现在有一个新项目想法。
先不要写代码，也不要先选技术。

请先帮我做需求澄清，输出：
1. 这个项目要解决的核心问题
2. 目标用户是谁
3. 核心使用场景是什么
4. 第一版最关键的价值是什么
5. 这个需求里目前最模糊、最容易失控的地方是什么
```

这一轮的目标：

- 把模糊想法变成清晰问题

---

## 第二轮：MVP 边界收敛

直接复制：

```text
基于刚才的需求澄清结果，请帮我收敛 MVP。

请输出：
1. 第一版必须有的功能
2. 可以第二阶段再做的功能
3. 暂时明确不做的内容
4. 为什么这样划分
```

这一轮的目标：

- 明确第一版做什么
- 明确第一版不做什么

---

## 第三轮：方案比较

直接复制：

```text
现在需求边界基本清楚了。
先不要写代码。

请基于当前 MVP，给我 2-3 种实现方案。
请从下面几个维度比较：
1. 开发速度
2. 复杂度
3. 维护成本
4. 风险
5. 扩展性

最后给我一个推荐方案，并说明为什么。
```

这一轮的目标：

- 不只要一个答案
- 让 Claude 讲清楚 tradeoff

---

## 第四轮：任务规划

直接复制：

```text
现在方案基本确定。
先不要写代码。

请把这个项目的第一阶段拆成 8-12 个小任务。
要求：
1. 每个任务尽量独立
2. 顺序合理
3. 标明依赖关系
4. 每个任务都能独立验证
5. 尽量适合小步实现和 review
```

这一轮的目标：

- 拿到可以执行的小任务列表

---

## 第五轮：生成项目规则 `CLAUDE.md`

直接复制：

```text
请根据我们刚才确定的需求、MVP、方案和任务规划，
生成这个项目的初始版 CLAUDE.md。

要求包含：
1. 项目目标
2. 当前阶段目标
3. 默认协作方式
4. 开发原则
5. 当前空白项目阶段的注意事项
```

这一轮的目标：

- 把前面讨论好的内容沉淀成长期规则

---

## 第六轮：建议最小 skills / agents 集合

直接复制：

```text
基于当前这个新项目的前期阶段，
请建议我最先建立哪些 skills 和 agents。

要求：
1. 只推荐最小必要集合
2. 说明每个的职责
3. 按投入低、收益高排序
```

这一轮的目标：

- 不过度配置
- 先建立最小协作框架

---

## 第七轮：进入实现前的确认

直接复制：

```text
现在请把前面这些结论整理成一个“进入开发前检查清单”。
要求包含：
1. 需求是否清楚
2. MVP 是否明确
3. 方案是否已选定
4. 任务是否已拆分
5. CLAUDE.md 是否准备好
6. skills / agents 是否有最小集合
```

这一轮的目标：

- 防止没准备好就开工

---

## 最后进入开发时的起始话术

当前面都完成后，再发：

```text
现在开始进入实现阶段。
请按我们已经确认的方案和任务顺序推进。
要求：
1. 每次只推进一个小任务
2. 尽量少改文件
3. 改完说明修改点、验证结果和残余风险
```

---

## 一句话总结

新项目里，Claude 最好的用法不是一上来就写代码，而是先帮你完成：

**需求澄清 -> MVP 收敛 -> 方案比较 -> 任务规划 -> 规则固化 -> 最小协作框架**
