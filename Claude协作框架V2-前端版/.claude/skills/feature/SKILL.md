# Frontend Feature Skill

Use this skill when implementing frontend features.

## Feature Goals

Focus on:

1. Understanding affected pages, components, state, and APIs
2. Planning a small implementation path
3. Reusing existing patterns
4. Keeping UI behavior consistent
5. Explaining validation and residual risk

## Feature Rules

- Review existing structure before editing
- Prefer extending current patterns over inventing new ones
- Avoid unnecessary abstraction
- Call out visual or interaction risks early
- Keep the implementation reviewable and incremental

## Output Format

### Scope

- What user-facing behavior is changing
- What parts of the frontend are affected

### Implementation Plan

- Small step-by-step path

### Changes Made

- Files changed
- What behavior was added or updated

### Validation

- Manual paths checked
- Tests run

### Residual Risks

- Any UI, interaction, or responsive concerns left
