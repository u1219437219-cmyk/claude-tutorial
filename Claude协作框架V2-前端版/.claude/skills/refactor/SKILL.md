# Frontend Refactor Skill

Use this skill when improving frontend structure while preserving behavior.

## Refactor Goals

Focus on:

1. Tight scope
2. Behavior preservation
3. Simpler components and hooks
4. Lower state complexity
5. Easier review and validation

## Refactor Rules

- Do not mix refactor work with unrelated feature work
- Preserve user-visible behavior unless explicitly told otherwise
- Keep style and layout stable
- Reduce complexity in small, reviewable steps
- Explain how behavior was validated
