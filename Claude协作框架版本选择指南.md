# Claude 协作框架版本选择指南

## 这三套分别是什么

桌面上现在有 3 套框架：

- [通用版](/Users/lfs/Desktop/Claude三件套模板)
- [前端版](/Users/lfs/Desktop/Claude协作框架V2-前端版)
- [后端版](/Users/lfs/Desktop/Claude协作框架V2-后端版)

它们都能用，区别在于关注重点不同。

---

## 先看结论

### 选通用版

适合：

- 项目刚起步
- 全栈混合项目
- 你还不确定主要问题更偏前端还是后端
- 你想先有一套稳妥基线，再慢慢细化

一句话：

不知道选哪个时，先选通用版。

### 选前端版

适合：

- 主要工作是页面、组件、交互、样式
- 项目以 React / Vue / Next.js / Nuxt 为主
- 你更担心 UI 回归、交互问题、状态混乱、响应式问题

一句话：

用户看得见的问题更多时，选前端版。

### 选后端版

适合：

- 主要工作是接口、服务、数据库、任务系统
- 项目以 API 服务、管理后台服务端、微服务为主
- 你更担心契约变化、数据风险、日志、并发、事务、缓存问题

一句话：

系统正确性和数据风险更重要时，选后端版。

---

## 怎么判断自己更偏哪边

### 如果你平时最常做的是这些

- 改页面
- 改组件
- 改样式
- 调交互
- 排查状态问题

选前端版。

### 如果你平时最常做的是这些

- 改接口
- 改服务逻辑
- 改数据库读写
- 查日志
- 排查任务或并发问题

选后端版。

### 如果你平时两边都碰，或者项目还在早期

选通用版。

---

## 三套的主要差异

### 通用版更强调

- 小步推进
- 最小修改
- 回归风险
- 测试缺口
- 可维护性

### 前端版更强调

- UI 回归
- 交互一致性
- 组件边界
- 状态复杂度
- 样式和响应式影响

### 后端版更强调

- 接口契约
- 数据一致性
- 错误处理和日志
- 并发 / 事务 / 缓存 / 消息风险
- 自动化验证

---

## 最稳妥的使用方式

### 方案 A：先通用，后细化

推荐大多数人这样用：

1. 先从通用版开始
2. 跑几轮真实协作
3. 发现问题明显偏前端，再切前端版
4. 发现问题明显偏后端，再切后端版

### 方案 B：从一开始就按项目类型选

如果你的项目类型很明确：

- 纯前端应用：直接选前端版
- 纯后端服务：直接选后端版

---

## 对团队的建议

如果团队里项目很多，推荐这样分层：

- 所有人先共用通用版理念
- 前端团队使用前端版
- 后端团队使用后端版

这样既统一方法，又保留针对性。

---

## 一句话最终建议

- 拿不准：先用通用版
- 偏页面和交互：用前端版
- 偏接口和数据：用后端版

最重要的不是一开始选得多完美，而是先用起来，再根据真实协作场景调整。
