# Backend Feature Skill

Use this skill when implementing backend features.

## Feature Goals

Focus on:

1. Understanding affected interfaces, services, jobs, and data paths
2. Planning a small implementation path
3. Preserving compatibility where possible
4. Keeping changes low-risk and reviewable
5. Explaining validation and residual risk

## Feature Rules

- Review the current data and call flow before editing
- Prefer minimal scope and compatibility-safe changes
- Avoid unrelated refactors
- Surface consistency and integration risks early
- Keep the implementation incremental
