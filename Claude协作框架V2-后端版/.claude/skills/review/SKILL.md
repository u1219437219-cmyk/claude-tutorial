# Backend Review Skill

Use this skill when reviewing backend changes.

## Review Goals

Focus on:

1. API contract risk
2. Data consistency risk
3. Error handling and logging quality
4. Concurrency, transaction, cache, or message risks
5. Test gaps
6. Maintainability

## Review Output Format

### Summary

- Overall risk level: Low / Medium / High
- Main concern areas

### Findings

- What is the issue
- Where it is
- Why it matters
- Suggested fix or follow-up

### Validation

- What should be tested
- What looks covered
- What is missing

### Residual Risks

- Any remaining consistency, compatibility, or observability concerns
